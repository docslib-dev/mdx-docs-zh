# 关于

本文简要阐述了MDX的诞生背景及其存在意义，
并向参与构建和启发这一项目的人们致谢。

{/* more */}

## 什么是MDX？

MDX是Markdown与JSX的结合体。
更多信息请参阅[§ 什么是MDX？][what]。

## 为什么选择MDX？

Markdown缺乏自定义组件的语法，
而MDX解决了这个问题。

虽然有许多语言在客观上优于markdown，
但markdown的独特优势在于：

* 直观呈现内容含义，具备良好的可读性
* 除[图片语法存在混淆][confusing]外，大多数场景下书写简便
* 宽松且容错性强：即使忘记书写星号，也不会报错（非常适合论坛评论场景）

Markdown*确实*提供了一种扩展方式——HTML，但存在以下局限：

* HTML在markdown中的解析逻辑不够智能
* HTML默认存在安全隐患，因此常被（部分）禁用
* HTML与markdown混合使用时规则混乱，例如空行或`markdown="1"`属性
* HTML与浏览器强耦合，而markdown的适用场景更广泛

前端领域提供了HTML的替代方案：JSX。
JSX的核心优势包括：

* 采用类XML的熟悉语法
* 语义无关性，专为编译器设计（可承载任意领域特定含义）
* 严格且无歧义（当作者遗漏斜杠时能及时报错，避免印刷书籍中出现错误内容）

## MDX受哪些项目启发？

以下项目、语言和文章在实现或理念层面影响了MDX的发展。

[JSX][]和[markdown][]语言是MDX的灵感来源。
Markdown由John Gruber（**[@gruber][gruber]**）创建，
而最流行的变体[CommonMark][]由John McFarlane（**[@jgm][jgm]**）等人开发。
JSX则由Facebook公司的Sebastian Markbåge（**[@sebmarkbage][sebmarkbage]**）等人设计。

当前`@mdx-js/*`项目深度依赖[unified][]（特别是[micromark][]、[remark][]和[rehype][]）以及[Acorn][]。
unified、remark、rehype及相关工具由Titus Wormer（**[@wooorm][wooorm]**）等人开发，
Acorn由Marijn Haverbeke（**[@marijnh][marijnh]**）等人创建。
