# 组件

MDX 的一大优势在于，您既可以编写 Markdown 语法，又能指定用自定义组件替代 HTML 元素。{/* more */}
下表列出了可被覆盖的 HTML 元素、对应的 Markdown 语法以及常规 JSX 等价写法。以首行的 `a` 元素为例，您可以通过以下方式将超链接替换为 `<FancyLink>`：

```tsx
import Readme from './readme.md' // 假设已使用集成工具将 MDX 编译为 JS
import {FancyLink} from './components/fancy-link.js'

<Readme components={{a: FancyLink}} />
```

更多 Markdown 使用指南请参阅 [CommonMark][]。

可覆盖的组件使用标准 HTML 元素名称。在 Markdown 中通常包括：`a`、`blockquote`、`br`、`code`、`em`、`h1`-`h6`、`hr`、`img`、`li`、`ol`、`p`、`pre`、`strong` 和 `ul`。通过 [`remark-gfm`][gfm]（[参见指南][guide-gfm]）还可支持：`del`、`input`、`section`、`sup`、`table`、`tbody`、`td`、`th`、`thead` 和 `tr`。其他声明支持 rehype 的 remark 插件所添加的新结构，同样适用于 MDX。

更多组件相关信息请参阅 [¶ 组件 in § 使用 MDX][components]。
